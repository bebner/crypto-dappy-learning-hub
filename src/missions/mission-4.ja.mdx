## ミッション概要

このミッションでは、Fungible Token と Flow のステーブルコインである FUSD に焦点を当てます。

このミッションの開始ブランチは [mission-4](https://github.com/bebner/crypto-dappy/tree/mission-4) です。

## ひとやすみ: スマートコントラクトの概要

このレッスンのトピックを始める前に、CryptoDappy を動かしているスマートコントラクトについてもっと知っておくといいかもしれません。この大まかな説明は、アプリケーションの全体的なアーキテクチャのメンタルモデル構築に役立ちます。

## Fungible Token: 大まかな概要

FUSD は Flow 上の Fungible Token 規格の実装であるため、FUSD を理解するには Fungible Token の概念をしっかりと理解する必要があります。

<YoutubeEmbed embedId="knfQSfkxBi0"/>

## FUSD: Vault と残高のチェック

大まかな概念を理解した後は、FUSD を実装するためのコードに飛び込んで、ユーザーの Vault（保管庫）と FUSD 残高をチェックしましょう。
